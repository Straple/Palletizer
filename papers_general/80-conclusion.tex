\Conclusion % заключение к отчёту

По результатам работы решены следующие задачи:
\begin{itemize}
  \item выполнена формальная постановка задачи трёхмерной паллетизации и введены необходимые метрические показатели: коэффициент заполнения (percolation) и минимальный коэффициент опоры (min\_support\_ratio);
  \item разработана и программно реализована вспомогательная структура данных HeightHandler для хранения карты высот с пятью различными реализациями (Baseline, Rects, Grid, SegTree, Quadtree);
  \item разработан жадный алгоритм укладки GreedySolver с ограничением на устойчивость размещения, достигающий коэффициента заполнения 56,04\% и минимального коэффициента опоры 30,31\% при среднем времени выполнения 11,1~мс;
  \item разработан метаэвристический алгоритм LNSSolver на основе метода поиска в большой окрестности (Large Neighborhood Search) с тринадцатью операторами мутации;
  \item проведено экспериментальное тестирование на 436 реальных задачах.
\end{itemize}

Экспериментальное сравнение пяти реализаций структуры HeightHandler показало, что простая реализация Rects на основе отсортированного списка прямоугольников превосходит более сложные структуры данных по производительности в 22 раза по сравнению с базовой реализацией.

Сравнительный анализ разработанных алгоритмов показал существенное превосходство метаэвристического подхода над жадным алгоритмом. Алгоритм LNSSolver при временном лимите 5 секунд достигает коэффициента заполнения 63,67\% и минимального коэффициента опоры 64,71\%, что превосходит результаты GreedySolver на 7,63 и 34,40 процентных пункта соответственно.

Введённое ограничение на положение центра масс коробок позволяет гарантировать устойчивость укладки: без данного ограничения коэффициент заполнения возрастает до 76,69\%, однако минимальный коэффициент опоры падает до 2,86\%, что неприемлемо для практического применения.

Разработанное программное решение может быть интегрировано в WMS-систему склада для автоматической генерации схем паллетизации. Время выполнения алгоритма LNSSolver порядка 5 секунд на паллету является приемлемым для задач складской логистики.

В дальнейшем планируется исследовать адаптивный выбор весов операторов мутации (Adaptive LNS) и обучение эвристики при помощи методов RL для online-сценариев.
