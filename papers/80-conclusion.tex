\chapter*{Заключение}
\addcontentsline{toc}{chapter}{Заключение}

В рамках данной работы была решена задача автоматизированной укладки коробок на паллету — актуальная проблема складской логистики, относящаяся к классу NP-трудных задач.

\section*{Основные результаты}

\begin{enumerate}
    \item \textbf{Разработана эффективная структура данных HeightHandler} для хранения карты высот паллеты. Сравнительное тестирование пяти различных реализаций (Baseline, Rects, Grid, SegTree, Quadtree) показало, что оптимизированный список прямоугольников (Rects) обеспечивает ускорение в 12.9 раза по сравнению с наивной реализацией.
    
    \item \textbf{Реализован жадный алгоритм GreedySolver}, который обеспечивает среднее заполнение 73.56\% при времени работы 308 мс на 436 тестов. Алгоритм подходит для систем реального времени и как базовое решение.
    
    \item \textbf{Разработан алгоритм LNSSolver} на основе поиска в большой окрестности (Large Neighborhood Search). Алгоритм достигает заполнения до 90.39\% при времени работы 300 секунд на тест, что соответствует уровню современных промышленных решений.
    
    \item \textbf{Реализованы метрики устойчивости укладки}: stability (перевязка слоёв), interlocking\_ratio (совпадение рёбер), center\_of\_mass\_z (высота центра масс). Метрики позволяют оценивать физическую устойчивость паллеты.
    
    \item \textbf{Создан интерактивный 3D-визуализатор} на Python с использованием Plotly. Визуализатор поддерживает пошаговый просмотр укладки, раскраску по SKU и генерирует автономные HTML-файлы.
\end{enumerate}

\section*{Практическая значимость}

Разработанное решение может применяться:
\begin{itemize}
    \item В системах управления складом (WMS) для автоматического формирования паллет
    \item В роботизированных складских комплексах для планирования укладки
    \item В логистических компаниях для оптимизации загрузки транспорта
\end{itemize}

Алгоритм LNSSolver с лимитом 1--5 секунд рекомендуется для систем реального времени, с лимитом 30--300 секунд — для офлайн-планирования.

\section*{Направления дальнейшей работы}

\begin{enumerate}
    \item \textbf{Многопаллетная укладка}: расширение алгоритма для одновременного планирования нескольких паллет с учётом последовательности отгрузки.
    
    \item \textbf{Учёт дополнительных ограничений}: хрупкость товаров, совместимость SKU, ограничения по весу на слой.
    
    \item \textbf{Интеграция с физическим симулятором}: валидация устойчивости укладки с помощью физического движка.
    
    \item \textbf{Машинное обучение}: использование RL-методов для обучения политики выбора операторов мутации.
    
    \item \textbf{Распараллеливание}: адаптация алгоритма для работы на GPU или в кластерной среде.
\end{enumerate}
