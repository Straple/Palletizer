## Реализованные солверы (`src/solvers/`)

| Класс | Идея | Литература / раздел |
|--------|------|---------------------|
| **`GreedySolver`** | Последовательная укладка: для очередной коробки из очереди паллеты перебираются ориентации и кандидатные точки с `HeightHandler`; выбирается позиция с минимальной итоговой высотой верха (tie-break по `x`, `y`). Очередь — порядок во входе после разбиения по паллетам. | Раздел «Жадный алгоритм» в `4-1-EYU.tex`; карта высот — там же, § HeightHandler. |
| **`DblfSolver`** | Вариант **Deepest Bottom Left Fill (DBLF)** для декодирования: среди допустимых позиций минимизируется лексикографически `(h_опоры, x, y, z_верха)` — сначала как можно ниже по высоте опоры, затем «нижний левый» угол, затем верх. Отличается от жадного при равных верхах. | DBLF в связке с ГА: Karabulut & İnceoğlu, 2004 `@karabulut`. Обзор 3D-эвристик: Zhao et al. `@Zhao2014`, Lodi–Martello–Vigo `@Lodi2002`. |
| **`FfdGreedySolver`** | **First-Fit Decreasing (FFD)**-препроцессинг: все экземпляры коробок сортируются по **убыванию объёма**, затем тем же жадным правилом, что и `GreedySolver`, раскладываются по паллетам (фиксированное разбиение по числу экземпляров на паллеты). | Классика для bin packing; 3D-варианты и эвристики — `@Martello2000`, `@Lodi2002`. |
| **`LNSSolver`** | **Large Neighborhood Search (LNS)** на кодировке «порядок + метаданные коробки»: мутации порядка/угла/поворота, симуляция — тот же жадный размещатель, приём только улучшающих шагов. | Pisinger `@Pisinger2005`; развитие ALNS — `@Ropke2006`; обзор — `@Pisinger2010`. В тексте ВКР: `4-1-EYU.tex`, § LNS. |
| **`GeneticSolver`** | Популяция тех же кодировок, элитизм, потомки от лучших особей + мутации `SimulationParams::mutate` (без скрещивания хромосом в текущей версии — эволюционная стратегия / «GA с мутацией»). | Общие идеи ГА для 3D: `@karabulut`, `@korelin`, `@Zhang2024`; связка GA+декодер — как у Karabulut (OBE+DBLF). |
| **`Solver` (базовый)** | Тривиальная заглушка для проверки конвейера метрик. | — |

## Алгоритмы из литературы (пока без отдельного класса в коде)

Их имеет смысл добавлять как следующий слой; ключи — из `pallete.bib`.

| Название | Суть | Статья / обзор |
|----------|------|----------------|
| **Skyline** | Профиль свободного пространства как «линия горизонта»; размещение в первую подходящую впадину, часто за `O(n log n)` в 2D. | Wei, Lim, Zhu, 2011 `@Wei2011`; обзор `@Zhao2014`. |
| **Maximal spaces** | Свободный объём как множество непересекающихся max-пространств; обновление после каждой коробки. | Parreno et al., 2008 `@Parreno2008`. |
| **Extreme Points** | Кандидатные точки на «экстремальных» границах уже поставленных блоков. | Crainic et al., TR `@Crainic2007`; упоминание в обзоре `@Zhao2014`. |
| **GRASP + локальный поиск** | Случайная конструкция + улучшение; для паллет с практическими ограничениями. | Calzavara et al., 2021 `@Calzavara2021`. |
| **Guided Local Search (GLS)** | Штрафы за «перегруженные» измерения, поэтапное смягчение. | Faroe, Pisinger, Zachariasen, 2003 `@Faroe2003`. |
| **Branch-and-bound + L2** | Точные/ветвь-и-граница подходы для малых `n`. | Martello, Pisinger, Vigo, 2000 `@Martello2000`. |
| **Guillotine / CP** | Ограничения гильотинных резов, constraint programming. | Amossen & Pisinger, 2010 `@Amossen2010`. |
| **RL / deep learning** | Политики для online 3D и strip packing. | Zhao et al., 2021 `@Zhao2021`; Fang et al., 2023 `@Fang2023`. |
| **Типология cutting & packing** | Классификация задач. | Wascher et al., 2007 `@Wascher2007`. |
